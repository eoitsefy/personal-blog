## 1. 当前产品形态

当前系统是一套面向个人日常记录的公开博客，采用“游客公开阅读、管理员登录创作”的模式。

当前没有开放注册入口，也没有普通注册用户的会话、个人中心或权限体系。数据库中的 `User` 主要用于管理员账号，应用实际只实现了管理员认证。

## 3. 已上线的文章管理功能

管理员登录后可以：

- 创建和编辑文章；
- 保存草稿、发布、撤回为草稿；
- 在发布前预览文章；
- 设置唯一 slug、标题、摘要和 Markdown 正文；
- 为文章设置一个分类；
- 为文章设置最多十个标签，并复用已有分类和标签；
- 按关键词、发布状态和删除状态筛选后台文章；
- 对后台文章列表分页；
- 将文章移入回收站；
- 从回收站恢复文章，恢复后的文章保持草稿状态；
- 对回收站文章执行带确认提示的永久删除。

文章管理已经包含服务端输入校验、公开可见性限制和媒体引用同步。

## 4. 已上线的图片与媒体库功能

目前的“媒体库”实际是图片媒体库，已实现以下能力：

- 仅允许已登录管理员上传；
- 支持 JPEG、PNG 和 WebP；
- 默认单文件最大 8 MiB，可通过环境变量调整，但服务端上限不超过 20 MiB；
- 校验文件声明 MIME、真实图片格式、扩展名、尺寸、像素数量和动画页数；
- 使用随机、不可预测的存储键，不直接使用上传文件名作为服务器路径；
- 持久化原始文件名、MIME、大小、宽高、SHA-256、所有者、公开 URL 和引用数量；
- 图片保存在持久化上传目录，通过 `/uploads/...` 提供稳定公开地址；
- 管理员可在媒体库查看图片、尺寸、大小、地址和引用数量；
- 支持分页、回收站、恢复和永久删除；
- 被文章引用的图片不能直接删除，避免文章出现失效资源；
- 可复制图片 Markdown，也可从文章编辑器中直接选择并插入图片；
- 删除文章或修改正文时会维护文章与图片之间的引用关系。

尚未支持普通附件、文档、压缩包、音频和视频文件，因此当前不能把图片媒体库描述为完整的“文件资产库”。

管理员认证使用数据库密码哈希和签名 HttpOnly Cookie。系统没有公开管理员注册接口；管理员账号通过受控命令创建或轮换密码。登录失败次数会被持久化限流，后台写操作包含来源、内容类型和请求大小检查。

## 6. 已完成的数据安全和运维能力

- PostgreSQL 保存文章、账号、分类、标签和媒体元数据；
- Redis 服务已随生产 Docker 环境运行，但当前核心文章流程不依赖它；
- Docker Compose 管理应用、PostgreSQL 和 Redis；
- Nginx 负责 HTTPS、反向代理和上传文件访问；
- 提供 `/api/healthz` 健康检查；
- 已建立部署前数据库备份、镜像回滚标记和部署日志；
- Phase 3A 生产部署已经验证首页、文章列表和真实文章详情路由；
- Phase 3B 已于 2026-07-17 部署到生产提交 `b767eb4`，并自动验收 canonical、Open Graph、JSON-LD、robots、sitemap、RSS、公开/非公开文章边界和安全响应头；
- 单元测试覆盖文章可见性、分类标签、搜索构造、认证、请求保护、图片验证、存储路径和媒体引用；
- PostgreSQL 集成测试覆盖登录、文章生命周期、图片上传、引用保护、恢复和永久删除。

## 7. 明确尚未完成的能力

以下能力目前不应对外宣称可用：

- 普通用户注册、登录、个人中心和用户管理；
- 通用文件上传与文件类型分类；
- 音频播放器、视频播放器或受控第三方视频嵌入；
- 地图展示、地点打点、轨迹或文章地点关联；
- 评论、回复、审核、举报和反垃圾；
- AI 文本问答、内容检索、语音输入、语音识别和语音播报；
- 浏览器端 Lighthouse 性能测量、完整键盘巡检和屏幕阅读器抽查。
