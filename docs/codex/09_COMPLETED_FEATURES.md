## 1. 当前产品形态

当前系统是一套面向个人日常记录的公开博客，采用“游客公开阅读、受邀用户登录、管理员创作与管理”的模式。

当前没有开放自助注册入口。管理员可以签发一次性邀请和密码重置链接；受邀用户完成注册后可以登录个人账户。账号体系具备普通用户/管理员角色、邮箱验证状态、账号停用和可撤销会话边界。

## 3. 已上线的文章管理功能

管理员登录后可以：

- 创建和编辑文章；
- 保存草稿、发布、撤回为草稿；
- 在发布前预览文章；
- 设置唯一 slug、标题、摘要和 Markdown 正文；
- 为文章设置一个分类；
- 为文章设置最多十个标签，并复用已有分类和标签；
- 按关键词、发布状态和删除状态筛选后台文章；
- 对后台文章列表分页；
- 将文章移入回收站；
- 从回收站恢复文章，恢复后的文章保持草稿状态；
- 对回收站文章执行带确认提示的永久删除。

文章管理已经包含服务端输入校验、公开可见性限制和媒体引用同步。

## 4. 已上线的通用媒体库功能

当前媒体库已经支持受控的图片、音频和文档资产，已实现以下能力：

- 仅允许已登录管理员上传；
- 图片支持 JPEG、PNG 和 WebP；音频支持 MP3、WAV、OGG/Opus；文档支持被动 PDF 与 UTF-8 TXT/Markdown；
- 默认单文件最大 8 MiB，可通过环境变量调整，但服务端上限不超过 20 MiB；
- 校验声明 MIME、扩展名与真实文件结构；图片还校验尺寸、像素数量和动画页数；
- 拒绝 HTML、SVG、Office、压缩包、可执行文件及包含明显脚本、附件或启动动作的 PDF；
- 使用随机、不可预测的存储键，不直接使用上传文件名作为服务器路径；
- 持久化资产类型、原始文件名、MIME、大小、宽高/时长、SHA-256、所有者、公开 URL 和引用数量；
- 允许的媒体文件保存在持久化上传目录，通过 `/uploads/...` 提供稳定公开地址；
- 管理员可查看资产类型、尺寸/时长、大小、地址和引用数量，并按类型、文件名、引用状态和删除状态筛选；
- 支持分页、回收站、恢复和永久删除；
- 被文章引用的资产不能直接删除，避免文章出现失效资源；
- 可复制图片、音频或文档 Markdown，也可从文章编辑器中直接选择并插入；
- 音频使用原生播放器；受控视频语法仅允许哔哩哔哩与 YouTube 正式 HTTPS 地址；
- 删除文章或修改正文时会维护文章与媒体之间的引用关系；
- 后台显示媒体总占用和 2 GiB 默认配额，达到 80% 时告警，超限上传由服务端拒绝；回收站文件永久删除前仍计入配额。

尚未支持自托管视频、Office、压缩包或任意附件；这些类型不会作为“其他文件”绕过白名单上传。

## 5. 当前账号与权限

| 能力 | 游客 | 注册用户 | 管理员 |
| --- | --- | --- | --- |
| 浏览已发布文章 | 可以 | 可以 | 可以 |
| 搜索、分类、标签和分页 | 可以 | 可以 | 可以 |
| 浏览草稿 | 不可以 | 不可以 | 可以预览 |
| 创建、编辑和发布文章 | 不可以 | 不可以 | 可以 |
| 上传和管理图片 | 不可以 | 不可以 | 可以 |
| 注册、登录个人账号 | 只能阅读登录入口 | 受邀注册、登录、退出和个人页已上线 | 管理员登录及用户管理已上线 |
| 发表评论 | 不能，游客只读 | 可以评论、一级回复、编辑/软删除本人评论和举报 | 可以评论并审核、隐藏、恢复、永久删除和锁定评论区 |

账号认证使用 bcrypt 密码哈希和数据库会话；浏览器保存 HttpOnly 原始令牌，数据库只保存 SHA-256 哈希。会话可以撤销，每个账号最多保留 5 个有效会话，停用账号会立即失去访问能力。邀请与密码重置令牌一次性使用、只保存哈希并带过期时间。系统没有公开自助注册或管理员注册接口。登录失败次数会被持久化限流，后台写操作包含来源、内容类型和请求大小检查。

评论系统还实现了每账号与来源组合限流、24 小时相同内容重复检测、长度/链接数量/控制字符/主动内容校验、纯文本展示，以及待审核、已发布、已隐藏和垃圾信息状态。管理员可审核、查看举报、软删除、恢复、永久删除和锁定单篇文章评论区。

## 6. 已完成的数据安全和运维能力

- PostgreSQL 保存文章、账号、分类、标签和媒体元数据；
- Redis 服务已随生产 Docker 环境运行，但当前核心文章流程不依赖它；
- Docker Compose 管理应用、PostgreSQL 和 Redis；
- Nginx 负责 HTTPS、反向代理和上传文件访问；
- 提供 `/api/healthz` 健康检查；
- 已建立部署前数据库备份、镜像回滚标记和部署日志；
- Phase 3A 生产部署已经验证首页、文章列表和真实文章详情路由；
- Phase 3B 已于 2026-07-17 部署到生产提交 `b767eb4`，并自动验收 canonical、Open Graph、JSON-LD、robots、sitemap、RSS、公开/非公开文章边界和安全响应头；
- 单元测试覆盖文章可见性、分类标签、搜索构造、认证、请求保护、图片验证、存储路径和媒体引用；
- PostgreSQL 集成测试覆盖登录、文章生命周期、图片上传、引用保护、恢复和永久删除。
- Phase 4B 评论系统已于 2026-07-18 部署到 `e51b390`；迁移、游客隔离、待审核可见性、审核发布、一级回复、举报、防重复、编辑重审、评论区锁定、回收站、恢复和永久删除均通过自动化及浏览器生产验收。

- 地图展示、地点打点、轨迹或文章地点关联；

- 浏览器端 Lighthouse 性能测量、完整键盘巡检和屏幕阅读器抽查。
