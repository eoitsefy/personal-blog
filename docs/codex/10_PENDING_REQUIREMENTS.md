# 待完成功能需求与实施规划

> 基准日期：2026-07-17
> 本文给出后续开发的默认产品决策、功能边界、依赖关系和验收标准。若产品所有者没有另行修改，可按本文默认方案实施。

## 1. 默认产品决策

### 1.1 浏览方式

采用以下默认规则：

- 已发布文章公开访问，不要求登录；
- 草稿、后台预览、文章管理和媒体管理仅管理员可访问；
- 软删除内容只在管理员回收站中可见；
- 暂不加入“仅注册用户可见文章”或付费内容；
- 后续如需要私人日记，新增 `PRIVATE` 或 `UNLISTED` 可见性，不能用“草稿”代替私人发布状态。

### 1.2 目标角色

| 角色 | 目标权限 |
| --- | --- |
| 游客 | 浏览、搜索、筛选文章，查看地图，播放公开音视频；可使用受限的 AI 文本/语音助手，但不能进入后台 |
| 注册用户 | 拥有游客全部能力；可发表评论、回复、管理自己的评论和举报不当内容 |
| 管理员 | 管理文章、分类、标签、媒体、地点、评论和用户；执行审核、删除、封禁及系统配置 |

注册用户只在评论功能开发时引入。个人博客不需要为了“存在一个角色”提前开放注册入口。

注册方式默认采用邮箱注册与邮箱验证；上线前必须具备限流、验证码或等效反自动化措施、密码重置和账号停用能力。若暂时无法可靠发送验证邮件，则先采用管理员邀请注册，不开放匿名自助注册。

### 1.3 评论权限

第一版评论只允许已验证的注册用户发表。游客可以阅读评论，但不能匿名提交。这样可以明显降低垃圾信息、冒用身份和隐私处理成本。

### 1.4 媒体策略

- 图片继续支持上传并复用；
- 通用文件和音频纳入统一媒体库；
- 视频第一版优先使用受信任来源嵌入，避免个人云服务器承担大文件存储、流量和转码；
- 不允许管理员在正文中直接写任意 `<iframe>`；应使用受控的媒体组件或自定义 Markdown 语法，并对来源做白名单校验；
- 当媒体体积或访问量明显增长时，将文件迁移到对象存储/CDN，应用层继续使用现有存储适配接口。

### 1.5 地图与位置隐私

- 文章可以关联零个或多个地点；
- 地点记录名称、说明、经纬度、发生时间、封面和关联文章；
- 内部保存统一坐标和原始来源信息，地图供应商通过适配层接入；
- 住宅、常驻地点或敏感地点默认只展示模糊坐标，不公开精确点位；
- 地图瓦片、坐标转换和公开展示必须符合实际部署地区及所选服务商的使用要求。

### 1.6 AI 助手边界

- AI 回答必须以已发布博客内容为知识来源并附文章引用；
- 先完成文本检索问答，再增加语音输入和语音播报；
- 游客可以使用，但必须有频率限制、每日额度、超时和总成本保护；
- 原始录音默认不长期保存；
- AI 或上游供应商不可用时，博客浏览、搜索和后台管理必须保持正常。

## 2. 功能需求清单

### 2.1 通用媒体库与文件复用

状态：核心范围已于 2026-07-17 部署，后续增强，优先级 P2。

生产环境已实现资产类型、音频真实格式校验、类型/文件名/引用状态筛选、文章音频引用和原生播放器；还实现了受控 PDF、UTF-8 TXT/Markdown 文档上传、文档复用与引用保护、后台存储用量展示和总配额拦截。HTML、SVG、Office、压缩包、可执行文件及带明显主动内容的 PDF 不在允许列表中。重复文件处理、上传时间筛选、对象存储迁移和其他文件类型仍属于后续增强。

需求：

- 将资产类型扩展为图片、音频、视频、文档和其他允许文件；
- 配置允许的 MIME、扩展名、单文件大小和总存储配额；
- 对文件名、真实内容、路径、重复文件和恶意文件进行服务端检查；
- 支持按类型、文件名、上传时间、引用状态和删除状态搜索筛选；
- 支持分页、详情、复制链接、下载、软删除、恢复和安全永久删除；
- 所有文章引用统一计数，禁止删除仍被引用的资产；
- 后台显示当前磁盘占用和接近配额的警告；
- 为对象存储迁移保留稳定 URL 或可批量迁移机制。

验收标准：

- 管理员可上传一份允许的文件，并在多篇文章中复用；
- 伪造 MIME、危险扩展名、路径穿越、超限文件和未授权请求均被拒绝；
- 被文章引用的文件不能被误删；
- 数据库记录和物理文件的删除、恢复流程一致。

### 2.2 音频与视频嵌入

状态：第一版已于 2026-07-17 部署，后续增强，优先级 P2。

本地音频上传与安全渲染已经实现；结构化视频语法仅接受哔哩哔哩和 YouTube 正式 HTTPS 地址，并固定转换为白名单播放器域名。短链、任意 iframe、非白名单域名和危险协议会被拒绝。自托管视频、字幕/文字稿字段和更完整的播放失败状态仍待后续切片。

第一版范围：

- 音频支持媒体库上传，并通过原生播放器播放；
- 音频组件提供标题、时长、播放、暂停、进度、音量和下载策略；
- 视频支持受信任来源的链接嵌入，并提供加载失败时的原始链接；
- 如允许小型自托管视频，仅支持明确白名单格式和大小，使用原生播放器，不在应用容器中实施视频转码；
- 编辑器通过按钮或结构化语法插入音视频，公开页面负责安全渲染；
- 禁止任意 HTML、脚本和任意 iframe 注入；
- 支持键盘操作、字幕/文字稿字段和移动端布局。

验收标准：

- 音频和允许的视频来源可以在文章预览和公开页正常播放；
- 不允许的域名、协议、格式和超限文件被拒绝；
- 播放器具备清晰的加载、失败和无障碍状态；
- 删除被正文引用的音视频资源时触发引用保护。

### 2.3 注册用户与权限系统

状态：邀请制第一版已于 2026-07-17 部署并验收，优先级 P1，是评论系统的前置条件。生产环境已经具备封闭角色枚举、账号启用/停用、邮箱验证状态、可撤销哈希数据库会话、管理员邀请注册、普通用户登录/账户页、一次性密码重置、用户停用与会话撤销。由于尚未配置邮件服务，自助注册保持关闭。

需求：

- 将角色定义为明确枚举，而不是任意字符串；
- 支持邮箱注册、验证、登录、退出、密码重置和账号停用；
- 管理员可查看、搜索、停用和恢复用户；
- 普通用户不能访问任何管理员页面或管理接口；
- 登录、注册、验证、重置和评论接口分别设置限流；
- Cookie、CSRF/来源校验、密码哈希和会话失效策略沿用并扩展现有安全基线；
- 提供隐私说明和最少必要的个人信息字段。

验收标准：

- 游客、注册用户和管理员的权限边界均有自动化测试；
- 停用用户无法继续登录或发表评论；
- 普通用户不能通过修改请求获得管理员权限；
- 邮箱验证与密码重置令牌一次性使用且按时过期。

### 2.4 评论系统

状态：第一版已于 2026-07-18 部署到生产提交 `e51b390` 并通过自动化及浏览器验收；后续增强优先级 P2。

生产环境已实现待审核评论、一级回复、本人编辑与软删除、管理员审核/隐藏/垃圾状态、举报、回收站、恢复/永久删除、单篇文章评论区锁定、持久化限流、重复内容检查、纯文本安全展示，以及评论规则与最少数据使用说明。自助邮件通知仍不在第一版范围内。

第一版范围：

- 已验证用户可对已发布文章发表评论；
- 支持一级回复，不做无限层级嵌套；
- 用户可编辑或软删除自己的评论；
- 管理员可审核、隐藏、恢复、永久删除和锁定评论区；
- 评论状态至少包含待审核、已发布、已隐藏和垃圾信息；
- 支持举报、频率限制、重复内容检测和基础反垃圾；
- 文章撤回或删除后停止新增评论；
- 前台提供加载、空状态、提交中、失败和审核中提示；
- 第一版可以不发送邮件通知，避免阻塞核心评论流程。

验收标准：

- 未登录用户不能提交评论；
- 用户只能修改自己的评论，管理员可以管理全部评论；
- 评论内容经过长度、链接数量和危险内容校验；
- 被隐藏或待审核评论不会作为公开内容返回；
- 文章、用户和评论删除策略不会产生孤立记录。

### 2.5 地图打点记录

状态：Phase 5A 与 Phase 5B 均已部署并完成生产验收；后续轨迹、路线与额度告警属于增强项，优先级 P2。

Phase 5A 已实现地点模型、四级隐私精度、坐标系与来源记录、图片封面引用、后台 CRUD/回收站、文章多地点关联、公开脱敏数据接口、地点搜索目录和无需第三方脚本的坐标概览。公开查询只接受未删除地点，并要求至少存在一篇已发布且未删除的关联文章；隐藏地点、草稿关系和内部坐标不会被序列化。

Phase 5B 已选择高德地图 JavaScript API 2.0，并实现真实底图、适配器、点位聚合、WGS84/BD09 转 GCJ-02、故障降级和受限运行事件日志。Web JS API Key 作为供应商规定的浏览器端公钥使用；`securityJsCode` 不进入仓库、Next.js 环境或浏览器，生产环境通过同源 Nginx `/_AMapService` 代理注入。

Phase 5B 已于 2026-07-18 部署到生产提交 `3395d9d4bc646e254423d3e27f7ccf68a7dc7160`。浏览器验收确认真实底图、地图控件、单点 Marker 和地点卡片跳转正常；供应商初始化返回 `OK/10000`，运行事件包含 `loader_ready` 和 `map_ready`，1 个公开点无遗漏。CSP 为高德 JS API 2.0 保留明确来源白名单并加入 `unsafe-eval` 兼容例外，后续可评估通过独立地图 iframe 缩小该例外的作用范围。

需求：

- `[Phase 5A]` 后台坐标表单支持新增、编辑、软删除、恢复和受保护的永久删除地点；
- 地点包含名称、说明、经纬度、发生时间、隐私精度、封面和关联文章；
- `[Phase 5A]` 文章编辑器可关联已有地点；从编辑器内直接创建地点留待交互增强；
- `[Phase 5A+5B]` 公开目录支持搜索、本地坐标概览、高德真实底图、点位聚合、详情卡片和跳转关联文章；
- `[Phase 5A]` 单篇文章展示允许公开的关联地点；
- `[Phase 5A]` 对敏感点位支持仅显示城市、管理员指定模糊坐标或完全隐藏；不使用每次请求随机抖动，避免位置漂移和缓存不一致；
- `[Phase 5A]` 无论地图脚本状态如何都提供可访问的地点文字列表；
- `[Phase 5B]` 地图服务只暴露受域名限制的 Web JS API Key；安全密钥由 Nginx 同源代理持有，额度告警在高德控制台配置。

验收标准：

- 管理员能创建地点并关联文章，游客能从地图进入公开文章；
- 草稿、软删除文章及私密地点不会通过地图泄露；
- 移动端可浏览，键盘用户可访问等价地点列表；
- 坐标来源、转换和展示精度有明确记录。

### 2.6 AI 文本与语音助手

状态：Phase 6A 基础已部署但保持关闭，优先级 P1/P2；文本检索、发布内容索引、引用校验、限流、每日额度、并发、超时、熔断和最小化用量日志已经完成禁用态生产验收。供应商已确定为 OpenAI，生成模型为 `gpt-5-nano`，向量模型为 `text-embedding-3-small`；当前待完成 Responses API 接入、服务器连通性、真实凭据、重建索引与启用态验收。Phase 6B 语音链路尚未开始。

按两个阶段实施：

1. 文本助手：抽取已发布文章、分块、索引、检索、生成带引用回答，并在发布、修改、撤回和删除时同步索引。
2. 语音助手：浏览器录音、语音转文字、可编辑文字问题、文本助手回答、文字转语音和播放控制。

共同要求：

- 模型、向量存储、语音识别和语音合成都通过可替换适配接口调用；
- 只检索已发布且未删除的内容；
- 没有证据时明确回答无法从博客内容确认，不编造来源；
- 返回文章标题、链接和相关片段；
- 防止提示词注入改变授权边界或泄露后台内容；
- 设置访客/IP/账号限流、每日额度、并发、超时和熔断；
- 记录用量、延迟和错误，但不记录密码、令牌或长期保存原始录音；
- 拒绝麦克风权限、浏览器不支持或供应商故障时可退回文字输入。

验收标准：

- 可对已发布文章提出文字和语音问题，并获得带有效链接的回答；
- 草稿、已删除文章和后台数据不会进入回答；
- 无证据、超额、超时、断网和供应商错误均有稳定且可理解的响应；
- AI 服务关闭后，博客其他功能不受影响。

## 5. 开始下一阶段前的确认项

以下默认值可以直接用于实施；只有产品所有者希望改变时才需要重新决策：

- 公开文章无需登录即可阅读；
- 注册用户只为评论等互动能力引入；
- 评论第一版不允许游客匿名发表；
- 音频可上传，视频优先使用受信任来源嵌入；
- 地点默认允许降低精度，敏感地点不公开精确坐标；
- AI 先文本、后语音，且只基于已发布文章回答；
- 用户生成内容、AI 和媒体功能上线前必须完成限流、审核、隐私和成本保护。
